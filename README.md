# college-budget-web

대학생 생활비 관리와 소비 판단을 위한 React + Vite 프로젝트입니다.  
백엔드 없이 `localStorage`만 사용하고, 월 수입을 기준으로 하루 예산을 계산한 뒤 실제로 이 소비를 해도 되는지를 빠르게 확인하는 데 초점을 둡니다.

## 1. 프로젝트 개요

이 프로젝트의 핵심 목표는 다음 한 문장입니다.

`지금 이 소비를 해도 되는지`를 빠르게 판단한다.

기록 중심 가계부보다, 오늘 쓸 수 있는 금액을 보여주고 지출이 가능한지 바로 확인하는 흐름을 우선합니다.

## 2. 현재 구현 범위

- 월 수입 입력과 저장
- 하루 예산 자동 계산
- 수동 하루 예산 덮어쓰기
- 전날 남은 금액 이월 사용 여부 관리
- 목표 저축액과 비상금 제외 설정
- 실시간 소비 가능 여부 판단
- 경고 메시지 표시
- 하루/월 기준 초과 알림과 재설정
- 지출 기록 저장
- 결제 수단 저장
- 고정지출 / 정기지출 타입 저장
- 템플릿 저장 및 불러오기
- 카테고리별 기본 통계
- localStorage 기반 로그인 상태

## 3. 페이지 구성

- `#/` 메인 화면
- `#/budget-status` 예산 상태
- `#/expense-records` 지출 기록
- `#/statistics` 통계
- `#/login` 로그인
- `#/my-page` 마이페이지

## 4. localStorage 사용

이 프로젝트는 `src/lib/storage.js`에서만 localStorage를 다룹니다.

사용하는 키:

- `monthlyIncome`
- `budgetAmount`
- `budgetSettings`
- `budgetAlertState`
- `expenseRecords`
- `expenseTemplates`
- `loginState`

## 5. 핵심 계산 로직

### 5.1 하루 예산 계산

- 자동 하루 예산 = 월 수입에서 목표 저축액과 비상금을 제외한 뒤, 이번 달 남은 일수로 나눈 값
- 수동 하루 예산이 있으면 자동 계산보다 우선합니다.
- 남은 금액 이월이 켜져 있으면 전날 실제로 남은 금액이 오늘 사용 가능 금액에 더해집니다.
- 월 수입이 비어 있거나 0 이하이면 하루 예산은 0으로 처리합니다.

### 5.2 소비 판단

- 입력 지출 금액이 현재 사용 가능 금액의 50% 미만이면 `안정`
- 입력 지출 금액이 50% 이상 75% 미만이면 `주의`
- 입력 지출 금액이 75% 이상 100% 미만이면 `경고`
- 입력 지출 금액이 현재 사용 가능 금액 이상이면 `초과`
- 판단 기준은 오늘 최종 사용 가능 금액의 비율입니다.

### 5.3 입력 예외 처리

- 빈값
- 숫자가 아닌 입력
- 0 이하 입력

### 5.4 초과 알림

- 하루 예산을 넘으면 하루 초과 배너를 표시합니다.
- 이번 달 사용 금액이 사용 가능 월 예산을 넘으면 월 초과 배너를 표시합니다.
- 초과 기준 재설정 버튼은 예산 숫자를 바꾸지 않고 현재 알림만 다시 계산하도록 만듭니다.

## 6. 지출 기록

- 지출 금액, 카테고리, 결제 수단, 지출 타입, 날짜, 메모를 저장합니다.
- 지출 금액은 숫자만 허용합니다.
- 저장된 지출은 바로 목록과 통계에 반영됩니다.
- 자주 쓰는 입력은 템플릿으로 저장하고 다시 불러올 수 있습니다.

## 7. 통계

- 저장된 지출 총액
- 기록 개수
- 카테고리별 지출 비중
- 지출 타입별 비중
- 결제 수단별 비중

## 8. 로그인

- 실제 인증이 아닌 UI 상태 저장입니다.
- 이름을 입력하면 로그인 상태가 저장됩니다.
- 마이페이지에서 현재 상태를 확인하고 로그아웃할 수 있습니다.

## 9. 실행 방법

```bash
npm install
npm run dev
```

빌드 확인:

```bash
npm run build
```
