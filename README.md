# college-budget-web

대학생 생활비와 지출을 localStorage 기반으로 관리하는 React + Vite 프로젝트입니다.  
백엔드, DB, 외부 API 없이 브라우저 안에서 예산 계산과 지출 기록, 통계를 확인할 수 있습니다.

## 1. 프로젝트 개요

이 프로젝트는 대학생이 월 수입과 하루 예산을 기준으로 지출을 관리하고, 카테고리별 소비 습관까지 확인할 수 있도록 만든 예산 관리 앱입니다.

핵심 목표는 다음과 같습니다.

- 월 수입 기준으로 하루 예산을 자동 계산
- 수동 예산 수정과 저장
- 남은 금액의 다음날 이월
- 지출 금액 판정과 경고
- 지출 기록, 정기 지출, 결제 수단 관리
- 통계와 절약 힌트 제공

## 2. 해결하려는 문제

대학생은 생활비가 한정되어 있고, 식비나 교통비처럼 자주 쓰는 지출이 많습니다.  
이 프로젝트는 다음 문제를 줄이기 위해 만들어졌습니다.

- 지금 얼마를 써도 되는지 바로 알기 어렵다
- 오늘 예산과 이번 달 예산을 따로 보기 어렵다
- 고정 지출과 정기 지출이 섞이면 관리가 복잡하다
- 지출 습관을 숫자로 보기 어렵다

## 3. 핵심 기능

### 3.1 하루 예산 자동 계산

- 월 수입을 기준으로 이번 달에 쓸 수 있는 금액을 계산합니다.
- 저축 목표 금액과 비상금 제외 금액을 먼저 뺀 뒤, 남은 금액을 남은 일수로 나눠 하루 예산을 계산합니다.
- `budgetSettings.manualDailyBudget`가 있으면 자동 계산값 대신 수동 예산을 사용합니다.

### 3.2 실시간 지출 판정

- `#/` 대시보드에서 지출 금액을 입력하고 판정할 수 있습니다.
- 입력값이 숫자가 아니면 `지출 금액은 숫자로 입력해 주세요.`를 표시합니다.
- 결과는 버튼을 눌러야만 보입니다.
- 예산 초과 시에는 판정 결과와 경고를 하나의 카드로 보여줍니다.

### 3.3 지출 기록 관리

- `#/expense-records`에서 지출 금액, 카테고리, 결제 수단, 날짜, 메모를 저장할 수 있습니다.
- 일반 지출, 고정 지출, 정기 지출을 구분해서 저장합니다.
- 정기 지출은 템플릿으로 따로 관리할 수 있고, 오늘 기록에 바로 추가할 수 있습니다.

### 3.4 예산 초과 경고

- 오늘 예산과 이번 달 예산을 각각 계산해서 경고 상태를 보여줍니다.
- 경고 기준은 `warningThreshold`로 조정할 수 있습니다.
- 예산이 초과되면 예산 설정 페이지로 이동해 기준을 다시 조정할 수 있습니다.

### 3.5 통계와 절약 힌트

- `#/statistics`에서 카테고리별 지출, 결제 수단별 지출, 날짜별 지출을 확인할 수 있습니다.
- 예산 대비 초과된 카테고리를 따로 보여줍니다.
- 학생 생활에 맞는 절약 힌트를 같이 제공합니다.

```text

├─ README.md
├─ package.json
└─ vite.config.js
```

## 7-1. 페이지 파일 구조 규칙

- 페이지 컴포넌트는 `src/pages/`에 둡니다.
- 파일명은 `PascalCase.jsx` 형식을 사용합니다.
- 공통 UI는 `src/components/`에 둡니다.
- 계산 로직과 저장소 접근은 `src/lib/`에 둡니다.
- 더미 데이터는 `src/data/`에 둡니다.

## 8. 라우팅 설계 규칙

이 프로젝트는 `HashRouter`를 사용합니다.

### 라우트

- `#/` - 대시보드
- `#/login` - 로그인
- `#/expense-records` - 지출 기록
- `#/budget-status` - 예산 설정
- `#/statistics` - 통계
- `#/my-page` - 마이페이지

## 9. 데이터 관리 방식

이 프로젝트는 백엔드 없이 `localStorage`만 사용합니다.

### 저장 원칙

- 저장소 접근은 `src/lib/`에서만 처리합니다.
- 페이지 컴포넌트는 저장소 함수를 호출해서 읽고 씁니다.
- 더미 데이터는 `src/data/`에 분리해서 둡니다.
- 숫자 입력과 계산 결과는 저장소를 통해 유지됩니다.

### 주요 localStorage 키

- `monthlyIncome`
- `budgetAmount`
- `budgetSettings`
- `expenseRecords`
- `recurringTemplates`
- `loginState`

### 데이터 구조

- 지출 기록은 배열 형태로 저장합니다.
- 각 기록은 `id`, `amount`, `category`, `date`, `memo`, `paymentMethod`, `kind`, `cycle`을 포함합니다.
- 예산 설정은 단일 객체로 저장합니다.

## 9-1. localStorage 설계 규칙

- 숫자형 설정은 가능한 한 숫자로 저장합니다.
- 로그인 상태는 별도 객체로 관리합니다.
- 지출 기록과 템플릿은 서로 다른 키로 분리합니다.

## 10. 핵심 계산 로직

### 하루 예산 계산

- `월 수입 - 저축 목표 - 비상금 제외 금액`으로 이번 달에 쓸 수 있는 금액을 구합니다.
- 남은 일수로 나눠 자동 하루 예산을 계산합니다.
- 수동 하루 예산이 있으면 자동 계산값을 덮어씁니다.

### 이월 계산

- 전날 남은 금액을 다음날로 넘길 수 있습니다.
- 이월 설정이 꺼져 있으면 기본 하루 예산만 사용합니다.

### 지출 판정

- 입력한 지출 금액이 오늘 사용 가능 금액보다 작으면 정상입니다.
- 경고 기준을 넘으면 주의 상태로 표시합니다.
- 오늘 예산을 넘으면 예산 초과 상태와 경고 문구를 함께 표시합니다.

### 예외 처리

- 빈값
- 숫자가 아닌 입력
- 0 이하 입력
- 예산 초과 상태

## 11. 로그인 기능 정의
