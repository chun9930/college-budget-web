# 디자인 복원 설계

## 목표
기능과 상태 관리는 그대로 유지하고, 현재 단순한 예산 관리 앱에 예전의 색감과 카드형 UI를 다시 입힌다.

## 접근 방식
- 로직 파일은 수정하지 않는다.
- 페이지 컴포넌트는 기존 기능 흐름을 유지한 채, 헤더/요약/폼 블록 정도만 카드형 구조로 정리한다.
- 스타일 복원은 `src/css/layout.css`, `src/css/pages.css`, `src/index.css` 중심으로 처리한다.
- 복원 대상은 시각적 위계, 카드 배경, 강조 색상, 상태 메시지, 레이아웃 간격이다.

## 범위
- 메인, 예산 상태, 지출 기록, 통계, 로그인, 마이페이지의 공통 톤 정리
- 상단 헤더와 네비게이션을 카드형 배경과 강조 색상으로 복원
- 상태 메시지를 더 분명한 성공/주의/오류 카드로 복원

## 제외
- 예전 고급 예산 기능 복원
- 정기 지출, 고급 통계, 고급 경고 로직 복원
- 상태 관리나 저장 구조 변경
